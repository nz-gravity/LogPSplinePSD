Multivariate blocked NUTS: math ↔ code#

This page documents the implemented multivariate likelihood and parameterisation used by the blocked sampler log_psplines.samplers.multivar.multivar_blocked_nuts.MultivarBlockedNUTSSampler.

It is intended as a durable bridge between the draft derivation in our paper

and the code under src/log_psplines/.

Overview#

The multivariate pipeline is:

  1. Build frequency-domain sufficient statistics from a multichannel time series using block-averaged FFTs.

  2. (Optional) coarse-grain the frequency axis by summing Wishart statistics across bins.

  3. Fit a Cholesky-parameterised spectral density matrix with P-splines, sampling each Cholesky row as an independent NumPyro/NUTS problem.

The implementation is designed around the Whittle/Wishart sufficient statistic

\[\Y(f_k)=\sum_{i=1}^{N_b} \underbrace{\d^{(i)}(f_k)\d^{(i)}(f_k)^*}_{\I^{(i)}(f_k)}\]

with degrees of freedom \(\nu\) equal to the number of non-overlapping blocks (called n_blocks in code).

Data → Wishart statistics#

The sufficient statistics are computed by log_psplines.datatypes.multivar.MultivarFFT.compute_wishart().

Given time-domain data x with shape (n_time, p), the code:

  • splits x into n_blocks contiguous non-overlapping blocks,

  • detrends each block (constant detrend),

  • applies a taper/window (default: Hann),

  • uses np.fft.rfft to work on the positive-frequency grid,

  • applies a one-sided PSD normalisation factor (Nyquist bin handled specially),

  • forms the per-frequency matrix

    \[Y(f_k) = \sum_{b=1}^{\nu} D_b(f_k)\, D_b(f_k)^H,\]

    where \(D_b(f_k)\) is the tapered/normalised FFT vector for block \(b\).

The code then eigendecomposes \(Y(f_k)\) and stores

\[Y(f_k) = U(f_k) U(f_k)^H,\]

where the columns of \(U(f_k)\) are \(u_\ell^{(k)} = \sqrt{\lambda_\ell^{(k)}}\, v_\ell^{(k)}\).

Those eigenvector-weighted components \(U(f_k)\) are the arrays u_re/u_im exposed on log_psplines.datatypes.multivar.MultivarFFT.

Cholesky parameterisation#

The blocked sampler follows the Rosen/Orihara-style inverse parameterisation:

\[S(f_k)^{-1} = T(f_k)^H\, D(f_k)^{-1}\, T(f_k),\]

where

  • \(D(f_k)\) is diagonal with entries \(\delta_j(f_k)^2\),

  • \(T(f_k)\) is unit lower-triangular with strictly-lower entries \(-\theta_{jl}(f_k)\).

In code, the diagonal field is represented as

\[\log \delta_j(f_k)^2 = \sum_m B_m(f_k)\, w^{(\delta)}_{j,m}\]

and each off-diagonal coefficient has its own real/imag spline field

\[\Re\,\theta_{jl}(f_k) = \sum_m B_m(f_k)\, w^{(\Re)}_{jl,m}, \qquad \Im\,\theta_{jl}(f_k) = \sum_m B_m(f_k)\, w^{(\Im)}_{jl,m}.\]

The blocked sampler actually samples distinct weight vectors per \(\theta_{jl}\) (within a fixed row \(j\)).

Per-row factorisation of the likelihood#

With the parameterisation above, the complex-Wishart/Whittle log-likelihood can be written as a sum of row-wise terms (one term per Cholesky row).

Let \(u_j(f_k)\) denote the \(j\)-th row of \(U(f_k)\), i.e. the vector of replicate components for channel \(j\) at frequency \(f_k\).

Define the row residual

\[r_j(f_k) = u_j(f_k) - \sum_{l<j} \theta_{jl}(f_k)\, u_l(f_k).\]

Exact per-row likelihood (from overleaf)#

The derivation expresses each factor as (verbatim LaTeX):

\[\begin{split}\mathcal{L}_j(\u_j,\u_{<j}|\btheta_j,\bdelta_j) \propto \nonumber \\ \prod_{k=1}^{N} \delta_{jk}^{-2N_b} \exp \left(\frac{-\sum_{\nu=1}^p\left|u_{j\nu}^{(k)}-\sum_{l=1}^{j-1}\theta_{jl}^{(k)}u_{l\nu}^{(k)} \right|^2}{T\delta_{jk}^2} \right)\end{split}\]

The blocked model implements (up to constants)

\[\log \mathcal{L}_j \;\propto\; -\nu\,\sum_k w_k\,\log\big(\delta_j(f_k)^2\big) \; - \sum_k \frac{\|r_j(f_k)\|_2^2}{\delta_j(f_k)^2}.\]

Key points:

  • \(\nu\) is the number of averaged blocks (fft_data.nu).

  • \(w_k\) are optional frequency weights (freq_weights). They are used to scale the log-determinant term when coarse graining is enabled.

  • The quadratic term uses the summed sufficient statistics directly, so it does not multiply by \(w_k\) again.

This is implemented in the internal NumPyro model _blocked_channel_model inside log_psplines.samplers.multivar.multivar_blocked_nuts.

Coarse-graining compatibility#

Coarse graining for multivariate FFT/Wishart statistics is performed by log_psplines.coarse_grain.multivar.coarse_grain_multivar_fft().

Within each coarse bin \(J_h\), it sums

\[\bar Y_h = \sum_{f\in J_h} Y(f)\]

and recomputes \(\bar U_h\) so that \(\bar Y_h = \bar U_h\bar U_h^H\).

The returned weights vector equals the bin member counts \(N_h\) and should be passed as freq_weights. With this choice, each bin behaves like a Wishart statistic with effective degrees of freedom \(\nu N_h\).

Exact coarse-grained likelihood form (from overleaf)#

The coarse-grained approximation is written as (verbatim LaTeX):

\[\prod_{h=1}^{N_c} \left|\S(\bar{f}_h)\right|^{-N_b*N_h} \exp\left(- \tr\left[ \S(\bar{f}_h)^{-1} \bar{\Y}_h\right]\right)\]

Symbol ↔ code mapping#

The table below lists the most important objects and where they appear.

  • \(p\) (number of channels) → fft_data.n_dim / self.n_channels

  • \(f_k\) (frequency grid) → fft_data.freq / self.freq

  • \(\nu\) (block count / Wishart DOF) → fft_data.nu / self.nu

  • \(U(f_k)\) (eigen replicates) → fft_data.u_re + i fft_data.u_im

  • \(\log \delta_j(f_k)^2\) → deterministic nodes log_delta_sq_{j}

  • \(\theta_{jl}(f_k)\) → deterministic nodes theta_re_{j}, theta_im_{j}

  • coarse-bin member counts \(N_h\) → fft_data.freq_bin_counts (stored)

  • likelihood weights \(w_k\) → config.freq_weights / self.freq_weights

PSD reconstruction#

Posterior samples of log_delta_sq and theta_re/theta_im are converted into a spectral density matrix by log_psplines.psplines.multivar_psplines.MultivariateLogPSplines.reconstruct_psd_matrix().

For each frequency, it builds \(T(f_k)\) with strictly-lower entries \(-\theta_{jl}(f_k)\), constructs \(D(f_k)=\mathrm{diag}(\delta_j(f_k)^2)\), and returns

\[S(f_k) = T(f_k)^{-1}\, D(f_k)\, T(f_k)^{-H}.\]

Notes on scaling conventions#

The derivation in overleaf writes the Whittle likelihood with an explicit observation-time factor \(T\). The exact LaTeX from overleaf is:

\[\mathcal{L}(\d|\S) \propto \prod_{k=1}^{N} \det(\S(f_k))^{-1} \times \exp\left(-\frac{1}{T}\d(f_k)^* \S(f_k)^{-1} \d(f_k)\right),\]

The implementation normalises the FFTs to one-sided PSD units during compute_wishart(). In that normalisation, the time/frequency-resolution factors are absorbed into the construction of \(Y(f_k)\) (and therefore \(U(f_k)\)). As a result, there is no explicit \(T\) factor in the NumPyro likelihood.

One way to see that this is consistent is to view it as a deterministic reparameterisation of the Fourier coefficients.

In one common Whittle convention (also stated in overleaf),

\[\d(f_k) \;\dot\sim\; \mathcal{CN}\!\left(0,\; T\,\S(f_k)\right),\]

which yields a quadratic term of the form

\[\exp\!\left(-\frac{1}{T}\, \d(f_k)^* \S(f_k)^{-1} \d(f_k)\right).\]

If instead we define the rescaled coefficient

\[ilde{\d}(f_k) = \frac{\d(f_k)}{\sqrt{T}},\]

then

\[ilde{\d}(f_k) \;\dot\sim\; \mathcal{CN}\!\left(0,\; \S(f_k)\right),\]

and the quadratic term becomes

\[\exp\!\left(-\tilde{\d}(f_k)^* \S(f_k)^{-1} \tilde{\d}(f_k)\right)\]

with no explicit \(T\).

The code effectively makes this kind of deterministic rescaling (along with the standard one-sided/Welch normalisation factors) when constructing \(Y(f_k)\) from the block FFTs. This is fine as long as you interpret \(S(f)\) in the implementation as being expressed in the corresponding one-sided PSD convention.

Exact Cholesky factorisation (from overleaf)#

The derivation in overleaf uses the following exact Cholesky parameterisation:

\[\mathbf{S}(f_k)^{-1} = \mathbf{T}_k^* \, \mathbf{D}_k^{-1} \, \mathbf{T}_k,\]

with

\[\begin{split}\bold{T}_k = \begin{pmatrix} 1 & 0 & 0 & \cdots & 0 \\ -\theta_{21}^{(k)} & 1 & 0 & \cdots & 0 \\ -\theta_{31}^{(k)} & -\theta_{32}^{(k)} & 1 & \ddots & \vdots \\ \vdots & \vdots & \ddots & \ddots & 0 \\ -\theta_{p1}^{(k)} & -\theta_{p2}^{(k)}& \cdots & -\theta_{p,p-1}^{(k)} & 1 \end{pmatrix}.\end{split}\]

The blocked sampler’s internal likelihood corresponds to the per-row likelihood factors \(\mathcal{L}_j\) (see Eq. (likelihoodj) in overleaf), modulo the implementation’s FFT/PSD normalisation conventions.
